Добавлена устойчивая обработка ошибок в ключевых сценариях:

1. Ошибки конфигурации и загрузки модели при старте (`MLFLOW_TRACKING_URI`, `DEFAULT_RUN_ID`, недоступный run) не роняют сервис.
2. При отсутствии загруженной модели `/predict` возвращает `503`.
3. Невалидный запрос и отсутствие обязательных фич корректно возвращают `422`.
4. Ошибки предобработки и инференса обрабатываются с `500` и логированием.
5. Некорректный `run_id` в `/updateModel` обрабатывается с `400`.

Добавлены тесты для:

1. Предобработки данных (`tests/test_features.py`).
2. Загрузки модели и доступа к признакам (`tests/test_model.py`).
3. Хэндлеров и валидации запросов (`tests/test_app_handlers.py`).
4. E2E-сценария запуска сервиса и предсказания (`tests/test_service_e2e.py`).

Реализовано логирование метрик в Graphite через StatsD:

1. Технические метрики: число запросов, время ответа, ошибки по кодам, CPU и память процесса.
2. Метрики данных: время предобработки, статистика по значениям фичей.
3. Метрики модели: время инференса, распределение вероятностей и предсказаний.
4. События модели: обновления модели, активный тип модели, список/число обязательных фичей.
